# How to query Kubepedia — indexes, facets, and entry points

## Summary

Kubepedia is AI-first but **not LLM-only**: a simple client — a shell script, a Telegram bot, a
full-text box — must be able to answer basic questions from `index/` alone, and richer clients layer
vector/graph retrieval on top. This page is the map of **how to find things**: which index file
answers which question, the four topical hubs, and how documents are addressed. It carries no domain
facts — it is the retrieval contract.

## Context

**The derived index (`index/`, regenerated by `scripts/generate_index.py`).** `kb/` is the source of
truth; `index/` is a deterministic projection a client reads instead of scanning Markdown:

| File | Shape | Answers |
|------|-------|---------|
| `documents.jsonl` | one row per doc: `id, type, title, status, *_version, aliases, tags, path` | "what docs exist / metadata of X" — enough to answer version questions **without opening the file** |
| `ids.txt` | every ID, sorted | existence / ID validation |
| `relations.jsonl` | `{source, type, target}` edges | graph traversal — "what relates to X", "what does X depend on" |
| `tags.jsonl` | `{tag, ids[]}` inverted | **facet:** "every doc tagged `dns` / `security` / `csi`" |
| `aliases.jsonl` | `{alias, ids[]}` inverted | **alias resolution:** a user's phrase → document IDs |

**Retrieval by client capability:**

- **No LLM (script/FTS):** resolve the query phrase against `aliases.jsonl`, or facet on
  `tags.jsonl`; read the hit's row in `documents.jsonl` for the versioned answer. Every doc's
  `## Summary` is written to stand alone as the short answer.
- **Graph client:** start from an ID and walk `relations.jsonl` (`depends_on`, `part_of`,
  `see_also`, …).
- **Vector/RAG client:** embed the doc bodies; the atomic one-fact-per-doc shape keeps chunks clean.

**The four topical hubs** (each is the curated entry point for its domain):

- [[CONCEPT-TROUBLESHOOTING_MAP]] — **symptom → doc**: an error/behaviour to the `TROUBLE-*` doc.
- [[CONCEPT-RUNBOOKS_INDEX]] — **operation → runbook**: an ordered procedure with rollback.
- [[CONCEPT-SECURITY_INDEX]] — **CVE / hardening / insecure-default** posture.
- [[CONCEPT-ADDON_CATALOG]] — the Helm/addon catalog.

**Addressing & versioning:**

- **IDs** are `TYPE-SLUG` (`COMPONENT-ETCD`, `TROUBLE-KUBEADM_UPGRADE_HEALTH_CHECK`); the prefix
  names the type, the ID is permanent across file moves and versions.
- **Version envelope** lives in frontmatter (`kubespray_version`, `kubernetes_version`,
  `component_version` as `>=vX <=vY`); **per-version specifics** live in the body (decisions.md
  D-017). A client filters candidates by envelope, then reads the body for the exact-tag detail.
- **Leaf docs** (`variable`, `ansible_tag`) are reached by **tag/alias facet and full-text**, not by
  graph edge (D-018) — so query `tags.jsonl`/`aliases.jsonl` for a variable, not `relations.jsonl`.
