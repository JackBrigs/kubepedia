# CoreDNS Corefile generated by Kubespray (plugins & forwarding)

## Summary

Kubespray renders the CoreDNS **Corefile** into a ConfigMap from
`coredns-config.yml.j2`. The main server block chains the standard plugins —
`kubernetes` (cluster records), `forward` (upstream), `cache`, `health`/`ready`,
`prometheus`, `loop`, `reload` — plus optional `hosts` and extra stub zones. Knowing the
generated layout tells you which inventory variable controls which plugin, so you edit
inventory rather than the live ConfigMap (which Kubespray overwrites).

## Configuration

**Main `.:53` block (plugins, in order):**

- `errors`, `health { lameduck … }`, `ready` — liveness/readiness.
- `kubernetes {{ dns_domain }}` — cluster DNS; extra domains via
  `coredns_kubernetes_extra_domains`, reverse zones (`in-addr.arpa ip6.arpa`) when
  `enable_coredns_reverse_dns_lookups`, extra options via `coredns_kubernetes_extra_opts`.
- `prometheus :9153` — metrics.
- `forward . <targets>` — **upstream resolution**: targets are `upstream_dns_servers` if
  set, else `/etc/resolv.conf`; extra forward options via
  `dns_upstream_forward_extra_opts` (map). This is the leg that resolves external names
  ([[TROUBLE-DNS_EXTERNAL_RESOLUTION]]).
- `cache` — from `coredns_default_zone_cache_block`.
- `loop` — self-reference guard (a misconfigured upstream trips it —
  [[TROUBLE-COREDNS_RESOLUTION_LOOP]]).
- `reload` — hot-reload the Corefile on change.
- `hosts /etc/coredns/hosts { … }` — present only when `dns_etchosts` is set (static
  records).

**Extra stub zones:** `coredns_external_zones` adds additional server blocks, each with
its own `forward . <nameservers>`, `cache`, `reload`, and optional `hosts` — for
delegating specific domains to specific resolvers.

## Compatibility

- Template stable across `v2.29.0`–`v2.31.0`; plugin set as above.
- The `forward` default (`/etc/resolv.conf`) means CoreDNS inherits the node's upstream
  resolvers unless you set `upstream_dns_servers` — the usual cause of external-resolution
  surprises.
- Do not hand-edit the `coredns` ConfigMap; change the variables and re-run so the
  template regenerates it (CoreDNS `reload` then picks it up).
