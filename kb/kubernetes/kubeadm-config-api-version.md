# kubeadm ClusterConfiguration API version generated by Kubespray

## Summary

Kubespray renders the kubeadm configuration it feeds to `kubeadm init`/`upgrade`
from a versioned template. Across `v2.29.0`–`v2.31.0` the effective format is
`kubeadm.k8s.io/v1beta4`. The mechanism that selects it changed in `v2.31.0`: the
`kubeadm_config_api_version` variable and the `v1beta3` fallback template were
removed, and the template is now pinned to `v1beta4`.

## Context

- Covers Kubespray `v2.29.0`–`v2.31.0`.
- The format tracks the kubeadm version, i.e. [[VARIABLE-KUBE_VERSION]]; kubeadm
  `v1beta4` is available for Kubernetes `>=1.31`.
- Supported Kubernetes windows per tag: see
  [[CONCEPT-KUBERNETES_VERSION_SUPPORT]].

## Configuration

In `v2.29.0`–`v2.30.0` the API version is a variable
(`roles/kubespray_defaults/defaults/main/main.yml:35`):

```yaml
kubeadm_config_api_version: "{{ 'v1beta4' if kube_version is version('1.31.0', '>=') else 'v1beta3' }}"
```

and the template is chosen dynamically
(`roles/kubernetes/control-plane/tasks/kubeadm-setup.yml:98`):

```yaml
src: "kubeadm-config.{{ kubeadm_config_api_version }}.yaml.j2"
```

Both `kubeadm-config.v1beta3.yaml.j2` and `kubeadm-config.v1beta4.yaml.j2` ship.
Because the minimum supported `kube_version` is already `>=1.31` in these tags
(`1.31.0` in v2.29.0, `1.32.0` in v2.30.0), the ternary always evaluates to
`v1beta4`; the `v1beta3` branch is unreachable.

In `v2.31.0` the indirection is removed: the variable is gone, the `v1beta3`
template is deleted, and `kubeadm-setup.yml:98` names the template directly:

```yaml
src: "kubeadm-config.v1beta4.yaml.j2"
```

Result per tag:

| Kubespray | Selection mechanism | Effective kubeadm API |
|-----------|---------------------|-----------------------|
| v2.29.0   | `kubeadm_config_api_version` variable (always resolves v1beta4) | kubeadm.k8s.io/v1beta4 |
| v2.29.1   | `kubeadm_config_api_version` variable (always resolves v1beta4) | kubeadm.k8s.io/v1beta4 |
| v2.30.0   | `kubeadm_config_api_version` variable (always resolves v1beta4) | kubeadm.k8s.io/v1beta4 |
| v2.31.0   | hardcoded template `kubeadm-config.v1beta4.yaml.j2` | kubeadm.k8s.io/v1beta4 |

## Compatibility

- Kubespray `v2.29.0`–`v2.31.0`: kubeadm config `v1beta4`.
- `v1beta3` remains selectable in code only in `v2.29.0`–`v2.30.0`, and only for
  `kube_version < 1.31`, which those releases do not allow — so it is effectively
  inert. It is fully removed in `v2.31.0`.
- kubeadm `v1beta4` requires Kubernetes `>=1.31`.
