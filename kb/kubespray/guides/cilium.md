# Configuring the Cilium CNI in Kubespray

## Summary
Covers how Kubespray deploys and tunes the Cilium CNI: IP Address Management (IPAM), the operator, kube-proxy replacement, transparent encryption (IPsec/Wireguard), BGP control plane, Hubble observability, and the timeouts that matter for large-cluster rolling upgrades. Key takeaway: most behavior is driven by `cilium_*` inventory variables, and several defaults (CIDRs, mask sizes, restart timeouts, CPU limits) inherit from cluster-wide values that need tuning at scale.

## Context
Applies when `kube_network_plugin: cilium`. Cilium is installed unprivileged by default (`securityContext.privileged: false`), which requires setting `kube_owner: root` in the inventory. Variables live in the inventory group_vars (e.g. `k8s-net-cilium.yml`) and are consumed by the `network_plugin/cilium` role.

## Implementation
- Unprivileged agent: set `kube_owner: root` (required with the default unprivileged securityContext).
- IPAM: default mode is "Cluster Scope"; select with `cilium_ipam_mode` (e.g. `cluster-pool`, `kubernetes`).
  - `cilium_pool_cidr` / `cilium_pool_cidr_ipv6`: cluster Pod CIDRs; default to `kube_pods_subnet` / `kube_pods_subnet_ipv6`. Warning: if the pool overlaps the node network range you lose inter-node connectivity.
  - `cilium_pool_mask_size` / `cilium_pool_mask_size_ipv6`: per-node CIDR size carved from the cluster pool; default to `kube_network_node_prefix` / `kube_network_node_prefix_ipv6`.
- Version: pin with `cilium_version` (e.g. `"1.19.3"`).
- Load Balancer IP pools: `cilium_loadbalancer_ip_pools` (list of named pools with `cidrs` and/or `ranges`).
- BGP control plane: enable with `cilium_enable_bgp_control_plane: true`.
  - New bgpv2 API (Cilium v1.16+): `cilium_bgp_cluster_configs`, `cilium_bgp_peer_configs`, `cilium_bgp_advertisements`, `cilium_bgp_node_config_overrides`.
  - Legacy (< v1.16): `cilium_bgp_peering_policies`.
- Kube-proxy replacement: set `cilium_kube_proxy_replacement` to `strict` (< v1.16) or `true` (v1.16+; `strict` is auto-converted to `true`). Cilium must know the apiserver address globally; only the localhost apiserver loadbalancer is usable in this mode, and only when it uses the same port as kube-apiserver.
- Cilium Operator (cluster-wide duties): extend with `cilium_operator_custom_args` (array or string), `cilium_operator_extra_volumes`, `cilium_operator_extra_volume_mounts`. Use `CILIUM_DEBUG` for debugging rather than a custom flag.
- Arbitrary config: `cilium_config_extra_vars` injects extra ConfigMap keys (e.g. `enable-endpoint-routes: true`).
- Identity allocation mode: "crd" (CRDs, `kubectl get ciliumid`) or "kvstore" (etcd).
- Transparent encryption: `cilium_encryption_enabled: true` plus `cilium_encryption_type` = `ipsec` or `wireguard`.
  - IPsec also needs `cilium_ipsec_key` (base64-encoded; not auto-generated by Kubespray — create per Cilium docs).
  - Wireguard requires Wireguard kernel mode on Linux 5.6+ (feature available Cilium 1.10.0+).
- Bandwidth Manager: `cilium_enable_bandwidth_manager: true`; requires Linux kernel 5.1.x+. Does not work with L7 network policies selecting the Pod at egress.
- Host Firewall: `cilium_enable_host_firewall: true` (disabled by default; can break cluster connectivity).
- Policy Audit Mode: `cilium_policy_audit_mode: true` — no policy is enforced; for validating impact only, not for production.
- Hubble: `cilium_enable_hubble`, `cilium_hubble_install`, `cilium_hubble_tls_generate`. Metrics via `cilium_enable_hubble_metrics` + `cilium_hubble_metrics` (dns, drop, tcp, flow, icmp, http). Access UI via `kubectl port-forward -n kube-system svc/hubble-ui 12000:80`.
- Upgrade / rolling-restart timeouts: BPF compilation makes DaemonSet pods slow to start. Tune `cilium_rolling_restart_wait_retries_count` and `cilium_rolling_restart_wait_retries_delay_seconds` so that count*delay >= number_of_nodes * per-pod start time. Default Cilium pod CPU requests/limits (100m:500m) are conservative and may need raising for faster startup.
