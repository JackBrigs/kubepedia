# Kubespray v2.27.0

## Summary

Kubespray **v2.27.0** ships Kubernetes **1.31.4** with the component set below. It is part of
the extended `v2.27.0`–`v2.31.0` coverage (baseline research is v2.29.0). kube-proxy nftables mode support added; Prometheus Operator CRDs installation added.

## Implementation

Component versions (tag `v2.27.0`):

| Component | Version |
|-----------|---------|
| Kubernetes default/min | 1.31.4 / 1.29.0 |
| etcd | 3.5.16 (K8s 1.29/1.30/1.31) |
| containerd | 1.7.24 |
| cri-o | per K8s minor |
| runc | 1.2.3 |
| cni-plugins | 1.4.0 |
| calico | 3.29.1 |
| cilium | 1.15.9 |
| CoreDNS | 1.11.3 |
| nerdctl | 1.7.7 |
| cri-dockerd | 0.3.11 |
| kube-vip | 0.8.0 |
| cert-manager | 1.15.3 |

## Upgrade Notes

- kube-proxy **nftables** mode support added; Prometheus Operator CRDs installation added.
- Multus bumped to 4.2.2; containerd static-binary install option; containerd 2.1.x hashes added.
- cri-o support on Ubuntu 20 dropped; Debian 13 support; certificate validity-period config (v1beta4).

## Compatibility

- Kubernetes line: **1.31.4** (minimum `1.29.0`).
- Component versions move per the selection mechanism ([[CONCEPT-COMPONENT_VERSION_SELECTION]]);
  concrete values above are for tag `v2.27.0`.
- Upgrade **one Kubespray minor at a time** ([[UPGRADE-KUBESPRAY_SEQUENTIAL]]).
