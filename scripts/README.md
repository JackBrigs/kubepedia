# scripts/

Служебные скрипты базы знаний.

Два поколения инструментов:

- **KDS-инструменты (0.2.0)** — работают с графом атомарных документов под `kb/`:
  `validate_kds.py`, `generate_index.py`, общий модуль `kdslib.py`, схема
  `schema/kds.schema.json`.
- **Легаси-валидатор (0.1.0)** — `validate_kb.py` проверяет навигацию старой
  версионной базы `knowledge-base/`, которая теперь хранится как сырой кэш
  источников (CLAUDE.md → «Baseline»). Оставлен для этого кэша.

---

## KDS-инструменты (0.2.0)

Зависимости: Python 3, **PyYAML**; для слоя проверки метаданных по схеме —
**jsonschema** (без него этот слой деградирует в предупреждение, остальные
работают).

```bash
pip install pyyaml jsonschema   # или в venv (PEP 668)
```

### generate_index.py

Генерирует производный машиночитаемый индекс из документов `kb/` (источник
истины) в `index/`:

```bash
python3 scripts/generate_index.py
```

- `index/documents.jsonl` — по объекту на документ (`id, type, title, status,
  версии, tags, path`);
- `index/relations.jsonl` — по объекту на типизированное ребро (`source, type,
  target`);
- `index/ids.txt` — все ID, отсортированы.

Индекс не редактируется руками — только регенерируется.

### validate_kds.py

Проверяет документы `kb/` по `standards/validation.md` (см. `standards/kds.md`
как единственный источник схемы). Слои: структура и профиль секций по типу,
метаданные по `schema/kds.schema.json`, грамматика и уникальность ID, соответствие
префикса ID типу, nullability версий по типу, наличие источников, резолв
типизированных связей, консистентность индекса (`index/` = свежий пересчёт).

```bash
python3 scripts/generate_index.py   # сначала обновить индекс
python3 scripts/validate_kds.py      # затем проверить
```

Код возврата: `0` — все жёсткие проверки пройдены; `1` — есть нарушения;
`2` — не найдена папка `kb/`.

### kdslib.py

Общий модуль: парсинг frontmatter/секций, таблицы (префиксы ID, профили секций,
правила версий), построение индекса. Держать в синхроне с `standards/kds.md` при
изменении KDS.

## validate_kb.py — валидатор консистентности (легаси 0.1.0)

Проверяет, что навигационный слой (`INDEX.md`, `README`-MOC) не разошёлся с
файловой системой — источником истины о составе базы (CLAUDE.md, раздел 11).

### Запуск

```bash
python3 scripts/validate_kb.py               # база по умолчанию: ../knowledge-base
python3 scripts/validate_kb.py path/to/kb    # явный путь
```

Код возврата: `0` — все жёсткие проверки пройдены; `1` — есть расхождения;
`2` — не найдена папка knowledge-base.

### Зависимости

- Python 3.
- **PyYAML** — нужен для проверки №4 (пересчёт счётчиков). Без него проверка
  пропускается с предупреждением, остальные выполняются.

```bash
pip install pyyaml     # или в venv, если система блокирует установку (PEP 668)
```

### Что проверяется

| № | Проверка | Жёсткая |
|---|---|---|
| 1 | каждая `versions/*/` упомянута в INDEX и наоборот (нет пропущенных/фантомных) | да |
| 2 | каждый `diffs/*.md` слинкован в INDEX и наоборот | да |
| 3 | число файлов `troubleshooting/issues/*.md` совпадает с заявленным в INDEX/README | да (эвристика) |
| 4 | счётчики в таблице статистики INDEX = пересчёт из YAML (переменные/inventory/теги) | да (нужен PyYAML) |
| 5 | все wiki-ссылки `[[...]]` резолвятся | да |
| 6 | `kubespray_version` во frontmatter файла `versions/<v>/` совпадает с папкой | да |
| 7 | у каждого YAML среза есть `kubespray_version` и `git_commit` (раздел 8) | да |

### Когда запускать

- на этапе 8 (Валидация) при добавлении версии;
- в ежевечернем прогоне (раздел 16);
- перед коммитом изменений в `knowledge-base/`.
