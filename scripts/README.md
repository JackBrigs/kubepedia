# scripts/

Служебные скрипты базы знаний (вне Obsidian-vault `knowledge-base/`, чтобы не засорять граф).

## validate_kb.py — валидатор консистентности

Проверяет, что навигационный слой (`INDEX.md`, `README`-MOC) не разошёлся с
файловой системой — источником истины о составе базы (CLAUDE.md, раздел 11).

### Запуск

```bash
python3 scripts/validate_kb.py               # база по умолчанию: ../knowledge-base
python3 scripts/validate_kb.py path/to/kb    # явный путь
```

Код возврата: `0` — все жёсткие проверки пройдены; `1` — есть расхождения;
`2` — не найдена папка knowledge-base.

### Зависимости

- Python 3.
- **PyYAML** — нужен для проверки №4 (пересчёт счётчиков). Без него проверка
  пропускается с предупреждением, остальные выполняются.

```bash
pip install pyyaml     # или в venv, если система блокирует установку (PEP 668)
```

### Что проверяется

| № | Проверка | Жёсткая |
|---|---|---|
| 1 | каждая `versions/*/` упомянута в INDEX и наоборот (нет пропущенных/фантомных) | да |
| 2 | каждый `diffs/*.md` слинкован в INDEX и наоборот | да |
| 3 | число файлов `troubleshooting/issues/*.md` совпадает с заявленным в INDEX/README | да (эвристика) |
| 4 | счётчики в таблице статистики INDEX = пересчёт из YAML (переменные/inventory/теги) | да (нужен PyYAML) |
| 5 | все wiki-ссылки `[[...]]` резолвятся | да |
| 6 | `kubespray_version` во frontmatter файла `versions/<v>/` совпадает с папкой | да |
| 7 | у каждого YAML среза есть `kubespray_version` и `git_commit` (раздел 8) | да |

### Когда запускать

- на этапе 8 (Валидация) при добавлении версии;
- в ежевечернем прогоне (раздел 16);
- перед коммитом изменений в `knowledge-base/`.
