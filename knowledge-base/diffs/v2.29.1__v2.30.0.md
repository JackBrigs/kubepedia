# Отчёт сравнения: v2.29.1 → v2.30.0

Сравнение соседних индексированных версий (раздел 7 CLAUDE.md). Основа — `git diff v2.29.1..v2.30.0` и сопоставление собранных срезов базы. Обновление между этими тегами — прямое и последовательное (без пропуска минорных версий), что соответствует официальной процедуре.

- **От:** `v2.29.1` (`0c6a295`, 2025-12-11)
- **К:** `v2.30.0` (`f4ccdb5`, 2026-01-30)
- **git diff:** 111 файлов изменено, +630 / −997 строк, 124 коммита между тегами.

## Что изменилось (обзор)

Минорный релиз: Kubernetes поднят до 1.34.x, обновлён стек контейнерных рантаймов и ряд аддонов, внесены **6 breaking changes**, удалён один Ansible-тег запуска (`master`). Основные риски обновления связаны с изменением источника endpoint для Cilium и удалением переменных `etcd_cert_dir_mode`, `runtime_engine`/`runtime_root`.

## Какие переменные добавлены

Реально добавленные переменные (по `git diff` defaults, не по счётчикам извлечения):

| Переменная | Файл | Назначение |
|---|---|---|
| `first_kube_control_plane` | `roles/kubespray_defaults/defaults/main/main.yml` | `{{ groups['kube_control_plane'] | first }}` — единая точка выбора «первого» control-plane (связано с breaking change по порядку узлов) |
| `kube_vip_address` | `roles/kubespray_defaults/defaults/main/main.yml` | адрес kube-vip (обновление роли kube-vip до 1.0.3) |

## Какие переменные удалены

| Переменная | Файл (v2.29.1) | Примечание |
|---|---|---|
| `etcd_cert_dir_mode` | `roles/etcd_defaults/defaults/main.yml` (было `"0700"`) | **breaking:** режим каталога сертификатов etcd теперь всегда `0700`, применяется нерекурсивно. Кто переопределял — сломается |
| `cilium_remove_old_resources` | `roles/network_plugin/cilium/defaults/main.yml` | удалена из defaults роли Cilium |

Дополнительно удалены ключи `runtime_engine` / `runtime_root` из конфигурации containerd (в шаблонах основного пути; легаси-шаблон `config-v1.toml.j2` для containerd < 2.0 может сохранять ссылки).

> Замечание о методике подсчёта: суммарные счётчики переменных сопоставимы — v2.29.1 — 1198, v2.30.0 — 1206. Фактические добавления/удаления берутся из `git diff` кода (выше), а не из разницы счётчиков. Реальное число top-level переменных в defaults `kubernetes-apps` практически одинаково (364 в v2.29.1, 365 в v2.30.0), что подтверждает: массовых удалений аддонов между версиями нет.

## Какие проблемы возможны при обновлении

### Breaking changes (требуют действий)

1. **Cilium endpoint.** `k8sServiceHost` / `k8sServicePort` теперь берутся из `kube_apiserver_global_endpoint` (`roles/network_plugin/cilium/templates/values.yaml.j2:10-11`). Нужно убедиться, что endpoint настроен и доступен со всех узлов. Раньше баг с захардкоженным IP первого control-plane ломал HA — см. [[troubleshooting/issues/cilium-hardcoded-apiserver-ip-ha-v2.30.0|разбор]].
2. **Порядок control-plane узлов.** Добавление нового control-plane «в начало» списка `kube_control_plane` больше не поддерживается — только в конец (введена переменная `first_kube_control_plane`).
3. **`etcd_cert_dir_mode` удалена** — режим всегда `0700`, нерекурсивно. Исправляет поломку Calico в режиме etcd datastore ([[troubleshooting/issues/etcd-cert-dir-recursive-perms-calico-v2.30.0|разбор]]), но ломает тех, кто переопределял переменную.
4. **`runtime_engine` / `runtime_root` удалены** из конфигурации containerd.
5. **`containerd_discard_unpacked_layers`** применяется только к containerd < 2.1.
