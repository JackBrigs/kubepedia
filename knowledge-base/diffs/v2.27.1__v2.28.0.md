# Отчёт сравнения: v2.27.1 → v2.28.0

Сравнение соседних индексированных версий (раздел 7 CLAUDE.md). Основа — `git diff v2.27.1..v2.28.0` и сопоставление срезов базы.

- **От:** `v2.27.1` (`45140b5`, 2025-06-27)
- **К:** `v2.28.0` (`63cdf87`, 2025-05-20)
- **git diff:** 525 файлов, +6310 / −42866 строк, 285 коммитов. Крупные удаления — вынос вендоренных данных и удаление ролей/провайдеров (Equinix, Krew, Heketi, contrib/kvm-setup, contrib/mitogen).

> Примечание по датам: тег v2.27.1 создан позже даты тега v2.28.0 по календарю (патч линии 2.27 выпущен параллельно после ветвления 2.28), но в цепочке апгрейда Kubespray v2.28.0 — следующий минорный за линией v2.27. Сравнение построено в порядке апгрейда.

## Что изменилось (обзор)

Крупный минорный релиз с масштабной реструктуризацией ролей и переходом на «underscore»-именование. Kubernetes поднят до 1.32, **containerd — до линии 2.0.x**, изменён механизм разрешения версий (из `checksums.yml` и словарей `*_supported_versions`). Удалён ряд провайдеров и вспомогательных инструментов. Заявлены будущие удаления (RHEL 8, Weave).

## Какие Ansible-теги запуска добавлены, удалены или изменили поведение

Всего тегов: 130 (v2.27.1) → **125** (v2.28.0).

- **Добавлен:** `bootstrap_os` (канонический тег роли `roles/bootstrap_os`, underscore).
- **Удалены:** `bootstrap-os` (дефисная форма как источник `facts`), `cephfs-provisioner`, `rbd-provisioner`, `cloud-provider` (переход на external cloud controllers), `krew`.
- **Изменили поведение/источник:** `always` (сбор фактов теперь через роль `network_facts`, плей `playbooks/facts.yml`); `facts` (источник сместился на `roles/bootstrap_os/*`); `system-packages` (перенесён в роль `bootstrap_os`, импорт `system_packages`); `gateway_api` (тег поднят на уровень `cluster.yml`); `kubelet-csr-approver` (сменился источник); `upgrade` (сократился на удалённые cephfs/rbd-провижинеры).

## Какие функции объявлены устаревшими

- **RHEL 8:** v2.28.0 — последняя версия с его поддержкой (PR #11872).
- **Weave:** анонсировано удаление в следующем релизе (в v2.28.0 ещё присутствует).
- `gateway_api_experimental_channel` — устарел (PR #11763).

## Какие проблемы возможны при обновлении

- **breaking (inventory):** убрать ведущую `v` из всех явных `*_version` в inventory (PR #11890) — иначе резолвинг версий сломается.
- **breaking (etcd):** удалён `etcd_kubeadm_enabled` — при его наличии в inventory задать топологию через `etcd_deployment_type`.
- **containerd 1.7 → 2.0:** мажорный переход рантайма; проверить совместимость конфигурации `config.toml` (v2 vs v3 схемы) и плагинов.
- **Удалённые провайдеры/роли:** Equinix, Heketi, Krew, cephfs/rbd-провижинеры — при использовании потребуется миграция.
- **Cilium через CLI:** изменился способ установки; проверить offline-реестр и доступность образа `cilium-cli`.
- **RHEL 8:** запланировать миграцию ОС до следующего релиза.
- Минорный шаг Kubernetes 1.31 → 1.32 выполнять строго последовательно, по стандартной процедуре upgrade-cluster.yml.
