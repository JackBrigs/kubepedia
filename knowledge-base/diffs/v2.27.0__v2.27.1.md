# Отчёт сравнения: v2.27.0 → v2.27.1

Сравнение соседних индексированных версий (раздел 7 CLAUDE.md). Основа — `git diff v2.27.0..v2.27.1` и сопоставление срезов базы.

- **От:** `v2.27.0` (`9ec9b3a`, 2025-01-02)
- **К:** `v2.27.1` (`45140b5`, 2025-06-27; Release опубликован 2025-07-28)
- **git diff:** 23 файла, +595 / −75 строк, 22 коммита.

## Что изменилось (обзор)

Патч-релиз линии v2.27. Новых breaking changes сверх v2.27.0 нет. Суть релиза: подъём патч-версий Kubernetes и компонентов (в т.ч. закрытие CVE-2025-1974 в ingress-nginx), одна новая переменная kubeadm, добавление нового sample-файла `kube_control_plane.yml` и точечные исправления в задачах control-plane/etcd. Набор Ansible-тегов запуска НЕ изменился (сверено: изменённые task-файлы не добавляют/не удаляют `tags:`).

## Какие переменные добавлены

| Переменная | Файл | Назначение |
|---|---|---|
| `kubeadm_image_pull_serial` | `roles/kubernetes/control-plane/defaults/main/main.yml` (`true`) | Управляет `imagePullSerial` kubeadm: последовательная (`true`) или параллельная загрузка образов. Проброшено в шаблон `kubeadm-config.v1beta4.yaml.j2`. |

## Какие переменные удалены

Из кода ролей — нет. В sample-inventory удалены **ошибочные** закомментированные примеры `kube_master_*` / `system_master_*` из `k8s_cluster/k8s-cluster.yml`: этих переменных никогда не было в коде ролей (сверено `grep` по `roles/` в обоих тегах). Корректное резервирование ресурсов control plane вынесено в новый файл `kube_control_plane.yml` через уже существующие `kube_*_reserved` / `system_*_reserved`.

## Какие значения по умолчанию изменены

- `kube_version`: `v1.31.4` → **`v1.31.9`**.
- Комментарий к `external_cloud_provider`: добавлен допустимый вариант `'manual'` (не устанавливает CCM Kubespray).

Изменений нет. Новых устареваний в патче нет (набор deprecated унаследован от v2.27.0).

## Какие проблемы возможны при обновлении

- Обновление в пределах патча линии v2.27 безопасно: минорная версия Kubernetes не меняется (1.31.x → 1.31.x), breaking changes отсутствуют.
- Исправления, вошедшие в v2.27.1 (по release notes): control-plane reconfiguration при upgrade, symlink'и сертификатов etcd, PodDisruptionBudget для CoreDNS, Calico `kubecontrollersconfigurations`, offline-скрипт под Podman. Тем, кто на v2.27.0, обновление до v2.27.1 рекомендовано (в т.ч. из-за CVE-2025-1974 в ingress-nginx).
- Если в inventory были скопированы несуществующие `kube_master_*` / `system_master_*` — они игнорировались и в v2.27.0; для резервирования ресурсов использовать `kube_*_reserved` / `system_*_reserved` (см. новый `kube_control_plane.yml`).
