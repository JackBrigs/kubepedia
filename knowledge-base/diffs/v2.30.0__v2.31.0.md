# Отчёт сравнения: v2.30.0 → v2.31.0

Сравнение соседних индексированных версий (раздел 7 CLAUDE.md). Основа — `git diff v2.30.0..v2.31.0` и сопоставление срезов базы.

- **От:** `v2.30.0` (`f4ccdb5`, 2026-01-30)
- **К:** `v2.31.0` (`1c9add4`, 2026-04-24)
- **git diff:** 210 файлов, +1816 / −4494 строк, 124 коммита. Крупные удаления — из-за выноса ролей ingress-nginx, Kubernetes Dashboard и Netchecker.

## Что изменилось (обзор)

Минорный релиз с самыми масштабными за три версии breaking changes: Kubernetes поднят до 1.35, **отключён cgroup v1 по умолчанию**, **удалены три аддона** (ingress-nginx, dashboard, netchecker), kubeadm-конфиг переведён только на v1beta4, etcd перешёл на линию **3.6.x**. Все проблемы, затрагивавшие v2.30.0, в v2.31.0 исправлены (см. раздел troubleshooting ниже).

## Какие Ansible-теги запуска добавлены, удалены или изменили поведение

- **Удалены теги** `dashboard`, `ingress-nginx`, `netchecker` (вместе с ролями). Итого: **123 → 120**.
- Новых тегов нет. Диспетчеризация CNI (`network`) и движков (`container-engine`) переведена на динамический `include_role` с apply-тегами; роль `network_facts` теперь под тегом `always`. Ссылки: [[versions/v2.30.0/ansible-tags|ansible-tags v2.30.0]] ↔ [[versions/v2.31.0/ansible-tags|ansible-tags v2.31.0]].

## Какие функции объявлены устаревшими / удалены

- **cgroup v1** — больше не поддерживается по умолчанию (K8s 1.35); `kubelet_fail_cgroup_v1: true`.
- **kubeadm v1beta3** — удалён, остался только v1beta4.
- **ingress-nginx, Kubernetes Dashboard, Netchecker** — удалены целиком.
- Роль валидации инвентаря теперь **прерывает плейбук** при обнаружении удалённых/устаревших переменных.
- Удалены: terraform-провайдер Nifcloud, документация RHEL 8.
- Требование Ansible: `>=2.18.0,<2.19.0` (Python 3.11–3.13); в v2.30.0 было `>=2.17.3`.

## Какие проблемы возможны при обновлении

### Breaking changes (требуют действий)

1. **cgroup v1.** На узлах с cgroup v1 обновление сломает kubelet — задать `kubelet_fail_cgroup_v1: false` либо мигрировать на cgroup v2.
2. **Удалённые переменные ломают плейбук.** Перед обновлением убрать из инвентаря `dashboard_enabled`, `ingress_nginx_enabled`, `deploy_netchecker`, `ingress_nginx_*`, `kubeadm_config_api_version` — роль валидации прерывает запуск при их наличии.
3. **Переименование bastion:** заменить `ssh_bastion_confing__name` на `ssh_bastion_config_name`.
4. **Уход с ingress-nginx/Dashboard/Netchecker** — заранее спланировать замену (внешний ingress-контроллер и т.п.).
5. **etcd 3.5.x → 3.6.x** — мажорное обновление etcd; выполнять строго через `upgrade-cluster.yml`.

### Процедура обновления

Строго последовательно: `v2.30.0 → v2.31.0`. Graceful — через `upgrade-cluster.yml`. Обязательна сверка инвентаря с sample и чистка удалённых переменных (иначе плейбук прервётся). Детали: [[versions/v2.31.0/docs/upgrades|дайджест обновления v2.31.0]].
