# Отчёт сравнения: v2.29.0 → v2.29.1

Сравнение соседних индексированных версий (раздел 7 CLAUDE.md). Основа — `git diff v2.29.0..v2.29.1` и сопоставление срезов базы. Этот отчёт стыкует вновь добавленную нижнюю цепочку (v2.27.0…v2.29.0) с ранее построенной частью базы (v2.29.1 → v2.30.0 → v2.31.0).

- **К:** `v2.29.1` (`0c6a295`, 2025-12-11)

## Что изменилось (обзор)

Патч-релиз линии v2.29. Новых breaking changes сверх v2.29.0 нет. Суть: подъём патч-версий Kubernetes и компонентов, консолидация переменной `unsafe_show_logs` (удалены дубли из defaults ролей). Набор Ansible-тегов (124), inventory и большинство docs не изменились.

## Какие переменные добавлены

Из кода ролей — новых пользовательских переменных нет.

## Какие переменные удалены

- `unsafe_show_logs` **удалена как дубль** из defaults ролей `bootstrap_os`, `etcd_defaults`, `kubernetes-apps/csi_driver/vsphere` (дедуп-рефакторинг). Переменная сохранена в единственном каноническом месте — `roles/kubespray_defaults/defaults/main/download.yml`, где её default изменён (см. ниже). В справочнике переменных это отражено уменьшением счётчиков: etcd 66 → 65, addons 367 → 366 (итого 1200 → 1198).

## Какие значения по умолчанию изменены

- `kube_version`: `1.33.5` → **`1.33.7`**.
- `unsafe_show_logs`: в v2.29.0 — литерал `false`; в v2.29.1 заменён на выражение, зависящее от CI-переменной окружения. **Следствие:** в v2.29.1 появляется расхождение формы sample↔defaults (в v2.29.0 его ещё нет) — зафиксировано в `discrepancies.md` обоих срезов.
- `cilium_hubble_envoy_image_tag`: обновлён (`v1.34.7-…` → `v1.34.10-…`).
