# Отчёт сравнения: v2.28.1 → v2.29.0

Сравнение соседних индексированных версий (раздел 7 CLAUDE.md). Основа — `git diff v2.28.1..v2.29.0` и разбор Release v2.29.0.

- **От:** `v2.28.1` (`a20891a`, 2025-08-26)
- **К:** `v2.29.0` (`9991412`, 2025-10-14)
- **git diff:** 269 файлов, +3279 / −3143 строк, 204 коммита.

## Что изменилось (обзор)

Минорный релиз. Kubernetes поднят до 1.33, **containerd — до линии 2.1.x**, **удалён CNI Weave**, переименован Ansible-тег `master` → `control-plane`, изменено поведение `/etc/hosts` и node-affinity CoreDNS. Появилась отдельная роль `etcd_defaults` (вынос defaults из роли `etcd`).

## Какие переменные добавлены

| Переменная | Назначение |
|---|---|
| `coredns_affinity` | Управление affinity подов CoreDNS (взамен удалённой захардкоженной node-affinity) |
| defaults роли `etcd_defaults` | etcd-переменные вынесены в отдельную роль `roles/etcd_defaults` (defaults + vars) |

Суммарно переменных: 1199 → **1200**.

## Какие переменные удалены

| Переменная | Причина |
|---|---|
| `conntrack_modules` | Убрано автоматическое заполнение `/etc/hosts` всеми узлами и связанная логика |
| `weave_*` (весь набор) | Удалён CNI Weave Net |

## Какие значения по умолчанию изменены

- `kube_version`: `1.32.8` → **`1.33.5`**.
- Поведение `/etc/hosts`: больше не заполняется записями всех узлов кластера (Urgent Upgrade Note).
- CoreDNS: удалена захардкоженная node-affinity, заменена настраиваемой `coredns_affinity`.

## Какие Ansible-теги запуска добавлены, удалены или изменили поведение

Всего тегов: 125 (v2.28.1) → **124** (v2.29.0).

- **Переименован:** `master` → **`control-plane`** (роль контрол-плейна в `cluster.yml`). Это breaking для тех, кто запускал `--tags master`.
- **Удалён:** `weave` (вместе с удалением CNI Weave Net).
- Прочие теги — без изменения поведения (проверено сопоставлением справочников v2.28.1 и v2.29.0/v2.29.1).

## Какие функции объявлены устаревшими / удалены

- **Weave Net** — удалён полностью (ранее анонсировался в v2.28.0).
- **CRI-O на Ubuntu 20.04** — поддержка прекращена.
- Заполнение `/etc/hosts` всеми узлами — удалено (Urgent Upgrade Note).

## Какие проблемы возможны при обновлении

- **breaking (Ansible-тег):** если в скриптах/CI используется `--tags master` — заменить на `--tags control-plane`.
- **breaking (Weave):** кластеры на `kube_network_plugin: weave` не обновятся — требуется миграция CNI до обновления.
- **breaking (/etc/hosts):** если инфраструктура полагалась на автозаполнение `/etc/hosts` всеми узлами — обеспечить резолвинг иначе (DNS); проверить `conntrack_modules` в inventory (удалён).
- **breaking (CoreDNS affinity):** ранее захардкоженная node-affinity удалена; при необходимости задать `coredns_affinity`.
- **containerd 2.0 → 2.1:** проверить совместимость конфигурации.
- **CRI-O + Ubuntu 20.04:** мигрировать ОС/рантайм.
- Минорный шаг Kubernetes 1.32 → 1.33 выполнять строго последовательно (upgrade-cluster.yml).
